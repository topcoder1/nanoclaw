# vscode-brain

Three commands that query your NanoClaw second-brain from inside VS Code:

| Command | What it does |
|---|---|
| **Brain: recall this symbol** | Recall on the identifier under the cursor (padded with the filename for context). |
| **Brain: recall selection** | Recall on the currently-selected text (capped at 500 chars). |
| **Brain: open this repo's journal** | Opens `<workspace-root>/docs/journal.md` (generated by `claw journal`). Offers to run `claw journal` if it doesn't exist yet. |

Results render in a side webview, ranked by `finalScore`. Click a repo hit to jump to the file at the right line; everything else opens the brain miniapp's KU page in your browser (where entities/chain/feedback live).

## Setup

```bash
cd tools/vscode-brain
npm install
npm run build
npx vsce package --no-dependencies        # produces vscode-brain-0.1.0.vsix
code --install-extension vscode-brain-0.1.0.vsix
```

The brain miniapp must be running (default `http://localhost:3847`). Loopback requests bypass the auth check, so no token is needed unless you point `brain.apiUrl` at a non-localhost host — in which case put `NANOCLAW_SERVICE_TOKEN` in `~/.claw/token`.

## Settings

| Key | Default | Notes |
|---|---|---|
| `brain.apiUrl` | `http://localhost:3847/api/brain` | Base URL of the miniapp's HTTP API. |
| `brain.serviceTokenPath` | `~/.claw/token` | Read once per request. |
| `brain.limit` | `10` | Hits to fetch. |
| `brain.scoreFloor` | `0.2` | Drop hits below this. Lower than agent auto-recall's 0.25 because the user is asking explicitly. |

## Why this surface specifically

- v1.3b shipped symbol-aware chunking, so a recall on `MyClass` lands inside the chunk that contains it (not a 200-line slab around it).
- v1.1b added `/api/brain/recall`, so this extension is a thin client — no new server work.
- v2 generated journals are already on disk, so "open this repo's journal" is one `showTextDocument` call.

## Click-through caveat

Repo hits use `source_ref = repo_name:relative/path.ts#L20-L40`. The extension turns `repo_name` into a workspace folder by basename match. If the repo isn't open as a workspace folder, you get a warning and the click is a no-op — the alternative would be to open the file outside any workspace context, which loses git, language servers, and everything else useful.
